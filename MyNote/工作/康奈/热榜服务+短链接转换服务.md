### konne-short-url

#### 项目结构

```
konne-short-url/
├── src/main/java/cn/konne/shorturl/
│   ├── config/           # 配置类（Auth、WebMvc）
│   ├── controller/
│   │   ├── ShortUrlController.java  # 生成短链接
│   │   └── VisitController.java     # 短链接跳转
│   ├── domain/           # 实体（ShortUrl、ShortUrlVo、SystemApiKeys等）
│   ├── interceptor/      # Token拦截器
│   ├── mapper/           # MyBatis Mapper
│   ├── utils/            # MmHashUtil、ShortCodeGenerator、UrlCheckUtil
│   └── KonneShortUrlApplication.java
└── src/main/resources/
    └── application.yaml
```

#### 核心功能

- **生成短链接**：长链接通过 MurmurHash64 去重，自增ID转短码，支持两种域名类型
- **短链接跳转**：302重定向，优先读 Redis 缓存，统计访问次数
- **中文URL处理**：自动对中文字符进行编码

#### 关键接口

| 接口 | 功能描述 | 请求方式 | 路径 |
|------|----------|----------|------|
| generateShortUrl | 生成短链接 | POST | /short/generate |
| redirect | 短链接跳转 | GET | /{shortCode} |

#### 技术实现

- **端口**：8080
- **短码生成**：自增ID → ShortCodeGenerator 转换
- **去重**：MurmurHash64 对原始URL哈希，避免重复生成
- **缓存**：Redis 缓存短链接，TTL 7天（`short_url:{shortCode}`）
- **域名**：type=1 → `w.wxb.cn/{shortCode}`，type=2 → `v.wxb.cn/{shortCode}`
- **数据库**：MySQL（short_url库）

#### 1.2 核心功能

- **热榜研判**：对获取的数据进行人工研判和处理，包括数据调性判断、所属地区、热榜分类等
    
- **机构配置（热榜开通）**：配置机构接收热榜数据范围、推送范围。
    
- **热榜模板**：提供个性化配置消息推送模板
    
- **热榜查询**：提供多维度查询热榜数据
    
- **预警消息**：查询已预警热榜消息及推送范围
    
- **操作日志**：记录已推送热榜查看情况
    

#### 1.3 关键接口

|   |   |   |   |
|---|---|---|---|
|接口|功能描述|请求方式|路径|
|page|热榜分页查询|POST|/hotList/page|
|page|预警分页查询|POST|/warnMsg/page|
|page|开通热榜机构分页查询|POST|/warnRule/page|
|getHotList|查询机构开通热榜数据（数解调用）|GET|/orgHot/getHotList/{orgId}|
|getCfgList|查询机构配置热榜（数解调用）|GET|/orgHot/getCfgList|
|getWarnList|查询机构热榜预警数据（数解调用）|POST|/orgHot/getWarnList|
|submitData|提交研判数据|POST|/judge/submitData|
|getData|获取研判数据|POST|/judge/getData|

#### 1.4 技术实现

- **数据存储**：使用 MySQL 存储结构化数据，Elasticsearch 存储搜索数据，日志。
    
- **消息队列**：使用 RabbitMQ 进行消息推送
    
- **缓存**：使用 Redis 进行缓存，提高查询性能
    
- **AI 集成**：集成 AI 模型进行热榜地区、调性、分类研判
    

#### 1.5 关键配置

- **服务端口**：8020
    
- **应用名称**：konne-hot-list
    
- **配置文件**：bootstrap.yaml，支持 dev、local、prod 环境

### 环境变量配置

  

各项目支持通过环境变量覆盖配置文件中的参数，主要环境变量包括：

  

- `SPRING_PROFILES_ACTIVE`：激活的环境配置（dev、local、prod）
    
- `SERVER_PORT`：服务端口
    
- `SPRING_DATASOURCE_URL`：数据库连接 URL
    
- `SPRING_DATASOURCE_USERNAME`：数据库用户名
    
- `SPRING_DATASOURCE_PASSWORD`：数据库密码
    
- `SPRING_REDIS_HOST`：Redis 主机地址
    
- `SPRING_REDIS_PORT`：Redis 端口
    
- `SPRING_RABBITMQ_HOST`：RabbitMQ 主机地址
    
- `SPRING_RABBITMQ_PORT`：RabbitMQ 端口
    
- `SPRING_ELASTICSEARCH_REST_URIS`：Elasticsearch 服务地址
    
- `SPRING_CLOUD_NACOS_CONFIG_SERVER_ADDR`：Nacos 配置中心地址
