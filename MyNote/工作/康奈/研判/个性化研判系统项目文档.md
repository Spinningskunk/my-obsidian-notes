
## 项目概述

本文档旨在对个性化研判系统的相关项目进行全面交接，包括项目结构、功能模块、技术栈、配置信息等。涉及的项目包括：

1. konne-data-judgment（https://gitee.com/konne/konne-data-review.git）
    
2. konne-judgment-count（https://gitee.com/konne/konne-data-review.git）
    
3. konne-data-judgment-conf（https://gitee.com/konne/konne-data-review.git）
    
4. konne-data-judgment-push（https://gitee.com/konne/konne-data-review.git）
    
5. konne-judgment-customer（https://gitee.com/konne/konne-data-review.git）
    
6. konne-judgment-task（https://gitee.com/konne/konne-data-review.git）
    
7. konne-three-summary（https://gitee.com/konne/kn-solution-sentiment.git）
    
      
    

## 项目架构总览

### 技术栈

|   |   |   |
|---|---|---|
|技术|版本|用途|

### 项目关系


各项目之间的关系如下：

- **konne-data-judgment**：核心研判服务，负责数据的获取、研判和处理
    
- **konne-judgment-count**：数据统计服务，负责统计研判数据
    
- **konne-data-judgment-conf**：配置管理服务，负责研判规则和配置的管理
    
- **konne-data-judgment-push**：推送服务，负责将研判结果推送给相关方
    
- **konne-judgment-customer**：客户服务，负责客户相关的研判数据处理
    
- **konne-judgment-task**：任务管理服务，负责研判任务的管理和分配
    
- **konne-three-summary**：三类研判汇总服务，负责汇总和管理三类研判数据
    

## 项目详细说明

### 1. konne-data-judgment

#### 1.1 项目结构

```Plain
konne-data-judgment/
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── cn/konne/data/judgment/
│   │   │       ├── config/           # 配置类（Redis、Elasticsearch、RabbitMQ、Kafka等）
│   │   │       ├── consume/          # 消息消费者（Kafka消息处理）
│   │   │       ├── controller/       # 控制器（数据研判、团队配置等）
│   │   │       ├── domain/           # 领域模型（数据实体、DTO、VO等）
│   │   │       ├── mapper/           # 数据访问层（MyBatis Mapper）
│   │   │       ├── service/          # 业务逻辑层（服务实现）
│   │   │       ├── utils/            # 工具类（Elasticsearch、ID生成等）
│   │   │       └── KonneDataJudgmentApplication.java  # 应用入口
│   │   └── resources/
│   │       ├── mapper/               # MyBatis 映射文件
│   │       ├── bootstrap.yaml        # 应用配置
│   │       └── database-sharding.yml # 分库分表配置
│   └── test/                         # 测试代码
└── pom.xml                           # Maven 依赖配置
```

  
#### 1.2 核心功能

- **数据研判**：对获取的数据进行人工研判和处理，包括数据调性判断、相关性分析、历史数据检测等
    
- **团队配置**：管理审核团队的配置信息，包括团队与方案的关联关系
    
- **数据查询**：提供多维度的数据查询接口，支持按媒体环节、媒体级别、调性等条件查询
    
- **自动审核**：支持自动审核功能，通过AI模型进行初步判断
    
- **消息消费**：消费 Kafka 消息进行数据处理
    
- **数据统计**：统计用户和团队的研判数据量
    
- **预警处理**：对符合预警条件的数据进行预警处理
    

#### 1.3 关键接口

|   |   |   |   |
|---|---|---|---|
|接口|功能描述|请求方式|路径|
|getUserProcessedData|获取用户已研判数据统计|GET|/data/getUserProcessedData|
|getPendingData|获取小组待处理数据统计|POST|/data/getPendingData|
|getPendReviewData|获取待研判数据列表|POST|/data/getPendReviewData|
|sumbitData|提交数据研判结果|POST|/data/sumbitData/{sumbitType}|
|wxDataQuery|研判预警数据详情查询|GET|/data/wxDataQuery|
|findAllDataRecord|查询指定数据操作记录|POST|/data/findAllDataRecord|
|addData|首页新增研判数据|POST|/data/addData|
|queryList|获取所有研判数据列表|POST|/data/queryList|

#### 1.4 技术实现

- **数据存储**：使用 MySQL 存储结构化数据，Elasticsearch 存储搜索数据，MongoDB 存储操作日志
    
- **消息队列**：使用 Kafka 进行数据传输，RabbitMQ 进行消息推送
    
- **缓存**：使用 Redis 进行缓存，提高查询性能
    
- **分库分表**：使用 ShardingSphere 进行分库分表，处理大数据量
    
- **AI 集成**：集成 AI 模型进行自动研判
    

#### 1.5 关键配置

- **服务端口**：8020
    
- **应用名称**：konne-data-judgment
    
- **配置文件**：bootstrap.yaml，支持 dev、local、prod 环境
    
- **分库分表**：使用 ShardingSphere 进行分库分表
